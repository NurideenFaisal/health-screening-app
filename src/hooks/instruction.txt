Build a multi-tenant, healthcare screening form system that removes all hardcoded forms and replaces them with a schema-driven template architecture. At the platform level, a superadmin creates reusable form templates that define the structure of a screening form, including sections and fields. Each template consists of multiple sections, and each section contains fields with types (text, number, select, etc.), validation rules, and optional configuration stored in JSON. These templates represent standardized clinical workflows (e.g., child screening, nutrition screening) and are globally reusable across clinics.

At the clinic level, clinic admins do not create forms from scratch. Instead, they select a superadmin-defined template and activate it per cycle. For each cycle, they assign clinicians either to full forms or specific sections using an existing assignment system. The frontend must dynamically render forms based on the selected template and enforce access control so clinicians only see assigned sections. The system must be fully cycle-aware, ensure data is stored in structured responses linked to form_id and field_id, and completely eliminate any hardcoded form UI or logic while maintaining compatibility with existing screening, children, and cycle workflows.
